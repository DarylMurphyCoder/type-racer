# Accessibility: screen reader & keyboard support

As a user with assistive needs, I want the app to be accessible (screen reader friendly, scalable text, keyboard-only control) so I can use the test.

Acceptance criteria

- Given a screen reader or keyboard-only navigation, when interacting with the test, then controls are reachable, readable labels exist, and color contrast meets WCAG guidelines.

Definition of done

- Accessibility review completed and any blocking issues fixed.

Priority: High
Estimate: 4 pts
Labels: user-story, priority:high
