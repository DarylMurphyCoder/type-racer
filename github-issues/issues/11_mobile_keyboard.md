# Mobile keyboard behavior

As a user on mobile, I want the virtual keyboard to behave well (no unexpected dismissals, correct focus) so typing is comfortable.

Acceptance criteria

- Given mobile device, when the test input is focused, then keyboard opens and stays visible while typing. Page scrolling should not hide input.

Definition of done

- Verified on Android and iOS webviews or mobile browsers.

Priority: High
Estimate: 2 pts
